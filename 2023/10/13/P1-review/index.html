P1教程部分

状态机编码风格

状态机的编码风格：一段式（高速状态机）、两段式、三段式。

一段式状态机

  • 整个状态机编写在一个always模块里
  • 同步时序逻辑，全部使用非阻塞赋值
  • 既描述状态转移，也描述状态的输入和输出
  • 维护代码和调试比较困难：这里解释一下

两段式状态机

  • 使用两个always模块，第一个always模块采用同步时序逻辑描述状态转移；；第二个always模块使用组合逻辑判断状态转移的条件，描述状态转移的规律。
  • 注意到第一个always模块使用的是时序逻辑，内部使用非阻塞赋值；而第二个模块是组合逻辑，内部使用阻塞赋值。

三段式状态机

  • 三段式状态机使用三个always模块，
  • 第一个always模块采用同步时序逻辑描述状态转移
  • 第二个always模块采用组合逻辑判断状态转移条件，描述状态转移规律
  • 第三个always模块采用组合逻辑或者时序逻辑描述每个状态的输出

表达式状态机

  • 是对整个字符串的判断，而不是只对后缀的判断，此处会影响状态转移
  • 注意clr信号，此处clr信号若为1，则要求state不能进行状态转移。注意使用else，这样就不会出现clr信号无效的情况

Verilog模块代码样例

  • MUX多路选择器，使用三目运算符实现
  • 如果设计有符号数的计算，不推荐使用三目运算符，而推荐使用always@(*)的方式,注意result此时应该为reg类型，注意此模板

P1课下答题

Q3 EXT扩展单元的实现

Q1与Q2较为简单，此处忽略，我们直接从Q3开始。
谁敢想笔者才开始竟然在那里傻傻的移位妄图做出来这个题！
实际上考察对位拼接运算符{}的使用，此处注意：

  • 表示重复多个字符，{16{imm[15]}}在16外层也有一层大括号，笔者在这里debug浪费了半个小时，因此最后如果是将imm进行符号扩展到32位，应当是{{16{imm[15]}},imm}，务必注意括号的个数以及匹配。
  • 此外，复习一个Q2提到的知识点，有符号数的算术右移$signed($signed(a)>>>b)

Q4 格雷码计数器

笔者的思路是设计一个中间变量count，仍采用十进制进行计数，最后使用mux多路选择器的模板对output进行选择输出。格雷码在卡诺图中用得较多。

Q5 表达式状态机

此题有以下几个关键点：

  • 注意审题，不是判断字符串后缀是否符合要求，而是整个字符串是否符合要求，这里对状态转移图有很大的影响
  • 到底什么是表达式F，注意它的递归定义，不仅仅是X * Y,X + Y,(X + Y) * Y,(X + Y) * X * Y,诸如此类均为表达式F
  • 异步清零，在Verilog中，同步复位只需在always模块的开头添加判断语句即可；对于异步清零，在原有的always模块的触发条件中加一个posedge clr,在模块内部写if(reset) begin state <= 0; end else begin 状态转移 end .注意加else！！！！不加的话就会出现原来貌似必须把判断语句放在最后的问题，但实际上只是自己逻辑上考虑不周到罢了

Q附加 BlockChecker

此题非常考验对状态转移的考虑，一不小心就会漏掉情况，下面提供笔者的思路

  • 首先，我们粗略的确认输入需要考虑的情况：空格、相关字母、其他。
  • 然后，我们先画一条主脉络
    空集->b->be->beg->begi->begin->begin ->begin e->begin en->begin end
  • 接着我们补充每个状态对于其他输入的情况。
  • 下面介绍笔者如何避免题目中的坑点：
    • begin必须出现在end之前：在状态中必须现有begin再有end，如果空串（或相当于空串，即前面的都匹配完成了）情况下直接输入end，则进入一个不会进行转移的状态，一旦为该状态，则输出为0
    • 允许嵌套：设置计数变量count，计算begin出现的次数，每当出现一次可以匹配的end，次数减一，输出时仅当count为0时才可能输出1；此外，当匹配完一次begin——end时，考虑到嵌套的存在，会根据当前count的大小判断是回到空串状态还是回到继续读取end的状态。
    • 无关输入全部转移到一个单独的状态。空格是偏离输出回到空串S0的唯一方法。
  • 异步复位仍然按照Q5的处理方法即可。
    PS：等腾出时间来笔者一定好好研究到底怎么放图，给各位展示我的状态转移图以供参考。

P1往年题

2021年

Q1 奇偶校验

  • 注意如果只对一个数字进行异或，那么就是其每一位进行异或，进而可以通过结果是0还是1判断出数据有奇数个还是偶数个1，是0则为偶数个1，是1则为奇数个1.
  • 注意，这里不能使用&去判断

Q2 心情电路

  • 注意在计数情况下的状态转换，比如某状态计数两个周期后进入其他状态，那么事实上当计数为1的时候就要进行状态转移了，因为本周期内还是该状态，下一周期开始时，刚好两个周期，刚好进行状态转移。注意，非阻塞赋值通常是在整个块结束后进行赋值。
  • 这种类型的题普遍有一个坑点，就是计数值的复位。此处只需牢记：一旦要进入需要计数的状态，就在此处对计数count清零，除此之外，在复位时也需要把所有计数值清零。
  • 异步复位的实现
    在always模块的敏感列表中加一个 posedge reset，内部加上判断即可.

方程识别

此题跟课下的表达式识别很相似。

  • 以等号为分界线，等号前和等号后的表达式判断虽然方法一样但是不可混用，否则无法判断是否已经是满足条件的方程式了。
  • 除此之外，其他的注意事项和课下的表达式判断相同。

一些零碎知识点的收获

  • integer类型也是寄存器数据类型，integer类型的变量为有符号数，而reg类型的变量则是无符号数

关于Verilog中的Moore型和Mealy型

看到一些学长的博客发现自己的Moore型状态机和模板并不一样，比如一般的Moore型模板会把输出写在always逻辑块中，而我通常是在最后assign。
实际上Moore和Mealy都可以用这种方式决定输出，唯一的不同在于Moore型的输出只与状态有关，而Mealy型的输出与状态和输入都有关。
因此，对于Verilog状态机，我们只需要分为三个模块initial,always,assign来写，其中initial负责初始化，always负责状态转移，assign负责最后的输出

字符串

  • 字符串保存在reg类型的变量中，注意一个字符占八个字节，因此寄存器变量的宽度应当足够大以保证不会溢出
  • 字符串不可多行书写，即不能出现回车
  • 如果寄存器变量的宽度大于字符串的大小，则使用0来填充左边的空余位
  • 如果寄存器变量的宽度小于字符串大小，则截去左边多余的数据
  • 在定义储存字符串的reg型变量时，必须使用小端，即reg [0:1023] data,而不是通常习惯的大端开头

使用字符串简化testbench

字符串自动机的testbench写着都很麻烦，因此参考学长的博客总结出一个简化版的写法

  1. 在testbench中定义一个1024位的寄存器，reg [0:1023] data ，在initial块中给data赋予你想测试的字符串。
  2. 根据上方第二条，寄存器中会使用0填充多余位置，因此使用while循环找到字符串存储的位置while(!data[0:7]) data = data << 8; #2 reset = 0; for(i = 0; i < 53; i = i + 1) begin in[7:0] = data[0:7]; data = data << 8; #2; end $finish,注意，这里循环中的53实际上是你需要测试数据的字符串长度，延迟长度为周期长度，使用时进行修改即可。或者使用
  3. 注意for循环的第三个部分，别手滑写成自增了，Verilog中没有自增。

always@(*)

always@(*)通常也是组合逻辑的一个写法，下面简单说一下笔者的理解（参考了网上的一些博客）

  • 敏感列表：该always块内所有输入信号的变化
  • 与assign的区别
    • assign是对wire型变量赋值，而always@(*)是对reg型变量进行处理
    • assign可以对变量赋定值，而always@(*)不能只赋定值，还需要这个值能发生变化，进而触发always块，如always@(*) begin temp <= 1'b0; end,由于常量一直不会发生变化，always块一直不会执行(参见第一大条的内容)，temp将一直处于一个不定态。

$signed()

这个知识点的原文章出处是roife学长的博客，笔者写在此处仅作为个人复习使用。
一个表达式中如果存在一个无符号数，那么整个表达式都会被当作无符号数。

  • 小数有符号，显式声明了进制的数无符号，除非用修饰符s声明其有符号
  • 位置选择，多位选择，位拼接的结果无符号（ b[2],b[3:4],{b}均无符号）
  • 比较表达式的结果无符号
  • 在assign加三目运算符的操作中，两个选择的数应当有无符号相同

位宽问题

阻塞与非阻塞赋值

  • 阻塞赋值：这里的阻塞指的是后面的语句必须在当前阻塞赋值语句结束后才能执行。并且，阻塞赋值是一步完成的：计算等号右边的值并同时赋给左边变量。
  • 非阻塞赋值：类比第一条，非阻塞赋值语句不会阻断其后的语句，且包含两个过程。
    • 首先计算等号右边的表达式的值.也就是说，在进入进程后，所有的非阻塞语句的右端表达式同时计算，赋值动作只发生在顺序执行到当前非阻塞语句的那一刻 注意，根据层次化事件队列，阻塞赋值和计算非阻塞赋值右边表达式的优先级相同，且相同优先级按顺序执行，因此本段代码中，b的值为1而非3
    • 一般来说，在执行完阻塞赋值和右侧表达式计算后，才真正更新左侧表达式的值。
