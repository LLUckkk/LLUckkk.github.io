实验一

网线制作

  • 网线——双绞线，8根铜线，每两根缠绕在一起
  • 相互缠绕的目的：铜线中电流产生电磁场的互相作用，以抵消临近线路和外界的干扰
  • 需要缠绕紧密
  • 直通线
  • 线序：橙白 橙 绿白 蓝 蓝白 绿 棕白 棕
  • 夹线钳的最上面是剥线刀口，中间是夹槽，下面的是剪线刀口
  • 金属片面向自己，从左到右是12345678
  • 先剥线，然后理线，把缠绕的分开，按照线序排列，用剪线刀口修齐
  • 然后将水晶头有弹片的一侧向下，用力将排好的线平行插入水晶头内的线槽中，并确保双绞线应插到线槽的顶端
  • 橙白线此时对着水晶头的从下面数的第一个引脚
  • 压线，水晶头放入夹槽中压线
  • 测试网线，将双绞线两端分别插入信号发射器和接收器，打开电源。如果网线制作成功，则1-8号指示灯亮起

报文分析软件

网卡的工作模式：广播、组播、直接、混杂四种模式

前两种模式下：网卡分别只能接收广播和组播报文，直接模式下。只能接受和自身硬件地址相匹配的单播报文；混杂模式：相连网络的所有报文

网卡通常缺省的工作模式是：广播模式，直接模式

简单组网实验

注意路由器和交换机的不同作用：

  • 路由器：负责不同子网之间的通信，工作在网络层

  • 交换机：在同一子网内通过MAC地址转发数据，无法跨子网通信，工作在数据链路层

实验步骤

  • 开启虚拟机，清空遗留的配置
  • 运行PCA桌面上的组网连线软件，按照组网图进行连线并提交
  • 在12进行的同时配置各台计算机的IP地址、子网掩码和默认网关
    • 本地连接——属性——Internet协议——使用下面的IP地址
  • 清空交换机的配置、路由器配置R1接口E0/0和E0/1的IP地址
    • 这个需要在R1的超级终端通过system-view进入系统视图
    • interface
  • 测试网络连通性，在命令行窗口，使用ping和ipconfig[/all]命令进行相应的操作
    • 在PCA的cmd确认连通性
    • ipconfig
    • 使用WireShark，本地连接开启抓包
    • 在PCApingPCB，ping PCC, ping PCD
  • 写实验报告

基于地址转换的组网实验

网络转换技术

原理：当A想向外网的B发送信息时，将目的IP和端口号以及信息发送给NAT路由器

路由器查找公网地址池，将本地的IP和本地的端口号转换为公网IP和端口号，并将这个对应关系记录到地址转换表上；然后将信息以及目的IP和端口号发送到Internet上

当到了B的时候，B将回复的报文和目的IP以及端口再发送给NAT路由器，此时的目的IP和端口是经过NAT转换后的，因此NAT会再次查看地址转换表将公网IP转换为本地IP，然后在发送到内部网络，最后发送给A

  • 上网地址选择192.168.20.100-200之间的地址
  • E0/1接口ip地址中的*=实验室号*15+组号*5+100
  • 每组分配的地址池范围：实验室号*15+组号*5+100-104

实验步骤

  • PCA上的组网连线软件连接好设备，注意设置一个PC监听R1

  • 先清除配置，设置各个计算机的ip地址、默认网关和DNS服务器

  • 配置路由器接口的IP地址

  • <Quidway> system
[Quidway] sysname R1
[R1] interface e0/0
[R1-...]ip address 192.168.2.1 24
[R1] interface e0/1
[R1-...]ip address 192.168.20.100 24

  • 配置访问控制列表

  • 配置地址池

  • 配置NAT转换
