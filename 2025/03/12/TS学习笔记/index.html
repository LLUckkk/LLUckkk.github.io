TS学习笔记

Created2025-03-12|Updated2025-03-15
|Post Views:

TypeScript学习

  • TS的静态检查机制给编写代码带来了极大的方便，把JS中很多运行时才能发现的错误在代码编写阶段就发现了

  • TS要编译为JS后才能运行

    手动编译：npx tsc <文件名>

    生成ts编译为js的配置文件：npx tsc --init，执行后会生成tsconfig.json

    全局观察npx tsc --watch，当ts文件改变时会自动编译成对应的js

    在config文件里把noEmitOnError的注释去掉，此时如果ts代码有错则不会重新编译。

  • 如果在Vue或者React中使用ts，则不需要自己编译

类型声明

类型推断

总的数据类型

  • ts包含js的所有数据类型

  • 同时新增了六个类型和两个自定义类型的方式

  • 一个注意点：类型首字母的大小写有区别

    • String：包装对象，复杂类型，内存中占用更多的空间
    • string：基元

    类似于java中int和Integer（感觉）

  • 自动装箱：

    比如：str.length，在执行的时候实际上先生成了一个包装对象，然后返回长度，销毁对象

常用类型

any

任意类型，一旦设置为any，则对该变量不会进行类型检查

any类型的变量可以赋值给任意类型的变量

unknown

类型安全的any

读取any类型数据的任何属性都不会报错，而unknown正好与之相反

never

不能有值，不限制变量，限制函数的返回类型，要么永远不结束，要么不正常结束

一般是ts主动推断出来

void

函数返回值的声明

object

小写object：所有非原始类型

大写Object：可以调用到Object方法的类型，所以不能存null和undefined

tuple

元组，特殊的数组类型，固定数量元素，每个元素类型已知且可以不同

不是关键字

enum

一组相关的值，定义一组命名常量

数字枚举

最后打印的时候，每个枚举和他对应的值都是相互映射的关系：DIRECTION.UP或者DIRECTION[0]

注意这里：每个枚举值的类型都是DIRECTION，也就是下面函数传参的时候只能传DIRECTION.xx，不能传DIRECTION

字符串枚举

此时就只有单方向的映射了

常量枚举

const关键字定义，编译时会被内联，避免生成一些额外的代码

将枚举成员的引用替换为他们的实际值，而不是生成额外的枚举对象，减少生成的JavaScript的代码量，提高性能

type

能为任意类型创建别名

这里的意思是：定义了一个新的类型，Status，可以是number也可以是string

用|连接的成为联合类型，表示一个值可以是几种不同类型之一

&交叉类型，相当于把两个类取并集

使用type声明函数的返回值为void的时候，也就是先声明再赋值，ts并不会严格要求函数返回空

声明

声明对象类型

属性后面加?表示赋值的时候这个属性有或没有都可以，但是不加的一定要有

索引签名

[key: string]: any

允许定义的对象具有任意数量的属性，键和类型可变

声明函数类型

声明数组类型

类

基本都是java里面的知识点

属性修饰符

类似java中的public、protected（类内部和子类）、private（仅类内部）

不加标识符默认public

多了一个readonly，只读，可以跟以上三个叠用

抽象类

抽象类不能被实例化，可以被继承，抽象类里可以有普通方法，也可以有抽象方法。

普通类继承了抽象类后必须实现抽象类中的抽象方法

抽象类的使用场景：

  • 定义通用接口，为一组类定义通用的属性、方法
  • 提供一些方法的基础实现，抽象类中定义的普通方法
  • 使用抽象方法确保关键实现，即继承了抽象类的类必须实现抽象方法
  • 共享代码和逻辑

接口interface

定义结构的一种方式，主要作用是为：类、对象、函数等规定一种契约，确保代码的一致性和类型安全

要注意接口只能定义格式，不能包含任何实现

接口的主流命名——I连接名字，名字加interface

接口之间可以继承

接口重复定义可合并

接口的使用场景：

  • 定义对象格式
  • 规定类的属性和方法
  • 自动合并

概念

interface vs type

都可以定义对象结构

不同点：

  • interface更专注于定义对象和类的结构，支持继承、合并
  • type：可以定义类型别名、联合类型、交叉类型，但不支持继承和自动合并，可以用&符实现类似效果，但是很不推荐

interface vs 抽象类

相同点：都能定义一个类的结构

不同点：接口只能描述结构，不能有任何实现代码，且一个类可以实现多个接口

抽象类：既可以包含抽象方法，又可以包含具体方法，一个类只能继承一个抽象类

泛型

泛型允许我们在定义函数、类或者接口的时候，使用类型参数来表示未指定的类型，这些参数在具体使用的时候才被指定具体的类型。

泛型能让同一段代码适用于多种类型，同时仍然保证类型的安全性

泛型接口

泛型还可以是复杂类型

通常以.d.ts作为扩展名，为现有的js文件提供类型信息，使得ts在使用这些js库或模块的时候可以进行类型检查或提示
